# BUG-001 — Graduation rejects active students

Priority:   Critical
Type:       Technical
Status:     Resolved
Scope:      Public
Found in:   deploy-2024.03.10.001
Reported:   2024-03-16
Resolved:   deploy-2024.03.16.002 (patch v15.1)

---

## Spec Reference

File:    spec/student/student_rules.dera
Section: Event graduate_student
Rule:    When academic = Active → Action perform_graduation

---

## Expected Behavior

A student with academic = Active who triggers graduate_student
should be routed to perform_graduation and receive:

  Outcome: Success
  Code:    STUDENT_GRADUATED

---

## Actual Behavior

System returns reject_request for all graduation attempts,
regardless of academic state.

---

## Type Classification

  Technical — the spec is correct, the code does not match the spec.
  Analyst AI not required. Builder AI fixes directly.

---

## Resolution

Builder AI identified a state comparison bug in the rules engine.
The Academic state value was being compared case-sensitive.
"Active" was stored as "ACTIVE" in the database.

Fix applied:
  patch/v15.1 → merged to main
  cherry-picked to req/REQ-045 (InTesting)
  cherry-picked to req/REQ-046 (InDev)

Deploy: deploy-2024.03.16.002
